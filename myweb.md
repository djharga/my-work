# تقرير مفصّل عن وضع المشروع — KHUTA React Platform

هذا التقرير حجْمِعْ مُحدَّث يشمل كل ما تم إنجازه في المشروع، ما المتبقي، الأخطاء الحالية والمحَلولة، وحالة البناء والتشغيل. الملف مُحدَّث بتاريخ العمل الأخير.

---

## 1) لمحة عامة سريعة
- المشروع: `khuty-react-platform` — واجهة React مبنية على Vite مع أنماط مركزية عبر `style.css` وواجهة مبنية لعرض دورات وتفاعل المستخدم.
- هدف التحديثات الأخيرة: توحيد أزرار الـ primary، إضافة خلفيات متحركة راقية، تحسين تفاعلات البطاقات، لوحة أوامر سريعة، وخيارات تحكّم لبيئة العرض (Demo Controls).

## 2) ما تم إنجازه (ملخّص مفصّل)
- بنية وتنظيم:
  - أنشئت/حدّثت `layouts/MainLayout.jsx` لاحتواء `Navbar`, `Footer`, و`DemoControls` (dev-only).
  - إعادة تنظيم `App.jsx` ليحتوي فقط على المسارات (Routes) داخل `MainLayout`.
- مكونات مرئية وتجربة مستخدم:
  - `Button.jsx`: توحيد أنماط الـ variants لاستخدام توكنات `khuta` (bg-khuta-primary, text-khuta-primary).
  - `AuroraBackground.jsx` + CSS (`animate-aurora-*`) لخلفية هيرو وCTA.
  - `GenerativeBackground.jsx` + أنماط CSS (`bg-radial-*`, `animate-gen-*`, `bg-noise`) — خلفية توليدية عالمية منخفضة التأثير (GPU-accelerated).
  - `ResponsiveTilt.jsx` ودمجه مع `CourseCard.jsx` و`FeatureCard` لإضافة تأثير ميلان 3D مع تعطيله على شاشات < 768px.
- أدوات إنتاجية:
  - لوحة أوامر (Command Palette):
    - `context/CommandPaletteContext.jsx` — مفتاح Ctrl/Cmd+K وEscape لإدارة الافتتاح.
    - `components/CommandPalette.jsx` — بحث قابل للبحث، نتائج مجمعة (Pages, Courses, Actions) ومعاينة ديناميكية للعناصر.
- حالة التفاعل والاختبار:
  - `DemoControls.jsx`: أدوات محاكاة (تسجيل دخول تجريبي، تسجيل خروج) ومفتاح تحكّم لتعطيل/إعادة تفعيل حركة الخلفية.
- سياق/حالة:
  - `BackgroundContext.jsx` تمت إضافته لتخزين حالة الخلفية مع حفظها في localStorage (khuta_generative_bg_enabled).
  - `CommandPalette` الآن يستمد قائمة الدورات مباشرةً من `CourseContext`.
- اعتمادات مثبتة:
  - `react-tilt`, `@headlessui/react`.

## 3) المهام المتبقية (موضوعة حسب الأولوية)
- ربط CommandPalette بترتيب أولويات ووزن نتائج البحث (ترتيب أبجدي أو عبر شعبية).
- إضافة اختصارات مرئية بجانب كل نتيجة (مثل ⌘K لفتح اللوحة أو مفاتيح مختصرة لكل نتيجة).
- تحسين معاينات النتائج: عرض صورة بديلة إذا غاب `course.image`، وأزرار سريعة داخل المعاينة (مثل Save/Share).
- تحسين أداء GenerativeBackground: خيار لتقليل جودة الضجيج أو تعطيله أو تبديل إلى canvas-based noise عند الحاجة.
- اختبار أتمتة (unit/integration) للمكونات الحرجة: `ResponsiveTilt`, `CommandPalette`, و`BackgroundContext`.
- تحسين الوصول (a11y): التأكد من تدرّج الألوان، تباين النص، وإمكانية التنقل باللوحة عبر لوحة المفاتيح فقط.

## 4) الأخطاء والمشكلات المحلولة
- Runtime: `[DemoControls] is not a <Route> component` — الحل: نقل `DemoControls` خارج `<Routes />` ثم دمجه داخل `MainLayout` أو عرضه كعنصر مستقل داخل `main.jsx` (تم حلّها عبر إعادة الهيكلة).
- Import error for `Tilt` — استيراد مع صحّة: `import { Tilt } from 'react-tilt'` ثم تغليفه في `ResponsiveTilt`؛ الآن يعمل البناء.
- تبعات Headless UI: ظهرت تحذيرات تحويلية أثناء البناء بخصوص `"use client"` داخل حزمة `@headlessui/react` عند التحزيم، لكنها لم تمنع البناء. (هذه تحذيرات غيابية على مستوى البندل الناتج — لا تأثير وظيفي مباشر الآن.)

## 5) الأخطاء/المشكلات الحالية (قيد المراقبة)
- تحذيرات أثناء bundling من مكتبة `@headlessui/react` تتكرر عند `npm run build` (ملاحظات حول `"use client"` في ملفات الحزمة) — لم تلحق خطأ تنفيذ لكن يجدر مراقبتها أو ترقية الحزمة/الإعدادات إذا ظهرت مشكلة لاحقًا.
- أداء GenerativeBackground على أجهزة منخفضة الموارد: الخلفية مُصمَّمة لتكون خفيفة لكن يحتاج فريق الاختبار إلى مراقبة استهلاك البطاقة الرسومية في الأجهزة الضعيفة.

## 6) حالة البناء والتكامل
- `npm install` و`npm run build` شُغّلا بنجاح بعد التعديلات المتتالية.
- لم تعد هناك أخطاء بناء تمنع الإخراج النهائي.

## 7) تعليمات تشغيل و مراجعة سريعة
1. تثبيت الحزم: `npm install`
2. تشغيل بيئة التطوير: `npm run dev`
3. فتح المتصفح على `http://localhost:5173` (أو المنفذ الظاهر في الـ terminal)
4. فتح اللوحة بواسطة: Ctrl+K (أو Cmd+K على macOS)
5. تعطيل/إعادة تفعيل الخلفية عبر: `Demo Controls` → "تعطيل حركة الخلفية" أو زر "إعادة تفعيل الخلفية".

## 8) خطوات مقترحة فورية (اختيارات سريعة)
- (1) ربط `CommandPalette` بترتيب نتائج ذكي (أولوية صفحات ثم دورات ثم إجراءات)، مع تسليط الضوء على الاختصار.
- (2) تحسين معاينة الدورات: إضافة زر "افتح" و"نسخ رابط" داخل المعاينة.
- (3) اختبار الأداء على جهاز محمول قديم والتقليل التلقائي لتأثيرات الخلفية عند تحميل كبير.
